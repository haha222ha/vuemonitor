## 2. 快速部署（Docker）

### 2.1 克隆项目

```bash
git clone https://github.com/xhs365/vuemonitor.git
cd vuemonitor
```

### 2.2 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env` 文件，**必须修改以下配置**：

```env
# ===== 必须修改 =====
JWT_SECRET=your-strong-random-secret-at-least-32-chars
JWT_REFRESH_SECRET=your-strong-refresh-secret-at-least-32-chars
ENCRYPTION_KEY=your-32-char-hex-encryption-key

DB_PASSWORD=your-strong-db-password
REDIS_PASSWORD=your-redis-password

# ===== 可选修改 =====
DB_HOST=postgres
DB_PORT=5432
DB_NAME=vuemonitor
DB_USER=saas_user
DB_POOL_SIZE=20

REDIS_HOST=redis
REDIS_PORT=6379

CORS_ORIGINS=["https://your-domain.com"]
ENVIRONMENT=production
DEBUG=false
LOG_LEVEL=info
LOG_FORMAT=json

DEEPSEEK_API_KEY=your-deepseek-api-key
```

### 2.3 生成安全密钥

```bash
# 生成 JWT Secret
python3 -c "import secrets; print(secrets.token_hex(32))"

# 生成加密密钥
python3 -c "import secrets; print(secrets.token_hex(16))"
```

### 2.4 启动服务

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看启动日志
docker-compose logs -f server

# 确认所有服务运行正常
docker-compose ps
```

### 2.5 初始化数据库

```bash
# 运行数据库迁移
docker-compose exec server alembic upgrade head
```

### 2.6 验证部署

```bash
# 健康检查
curl http://localhost:8000/health

# 预期返回
# {"status":"ok"}
```

## 3. 手动部署

### 3.1 安装 PostgreSQL

```bash
# Ubuntu
sudo apt install postgresql-16 postgresql-contrib-16

# 创建数据库和用户
sudo -u postgres psql
CREATE USER saas_user WITH PASSWORD 'your-strong-password';
CREATE DATABASE vuemonitor OWNER saas_user;
GRANT ALL PRIVILEGES ON DATABASE vuemonitor TO saas_user;
\q
```

### 3.2 安装 Redis

```bash
# Ubuntu
sudo apt install redis-server

# 配置密码
sudo vi /etc/redis/redis.conf
# 取消注释 requirepass 并设置密码
# requirepass your-redis-password

sudo systemctl restart redis-server
```

### 3.3 部署服务端

```bash
cd server

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件

# 运行数据库迁移
alembic upgrade head

# 启动服务
uvicorn app.main:app \
  --host 0.0.0.0 \
  --port 8000 \
  --workers 4 \
  --log-level info \
  --access-log
```

### 3.4 部署 Web 用户端

```bash
cd web-user

# 安装依赖
npm install

# 配置 API 地址
# 编辑 src/utils/api.ts，设置 baseURL

# 构建生产版本
npm run build

# 输出目录：dist/
# 使用 Nginx 托管静态文件
```

### 3.5 配置 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }

    # 健康检查
    location /health {
        proxy_pass http://127.0.0.1:8000;
    }

    # Metrics
    location /metrics {
        proxy_pass http://127.0.0.1:8000;
        allow 10.0.0.0/8;
        allow 172.16.0.0/12;
        deny all;
    }

    # Web 用户端静态文件
    location / {
        root /var/www/xhs365/web-user/dist;
        try_files $uri $uri/ /index.html;
    }
}
```

### 3.6 配置 Systemd 服务

```ini
# /etc/systemd/system/xhs365-server.service
[Unit]
Description=XHS365 API Server
After=network.target postgresql.service redis.service

[Service]
Type=simple
User=xhs365
Group=xhs365
WorkingDirectory=/opt/xhs365/server
Environment="PATH=/opt/xhs365/server/venv/bin"
ExecStart=/opt/xhs365/server/venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000 --workers 4 --log-level info
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable xhs365-server
sudo systemctl start xhs365-server
```

## 4. 生产环境部署

### 4.1 安全检查清单

- [ ] **密钥配置**：JWT_SECRET、JWT_REFRESH_SECRET、ENCRYPTION_KEY 已更换为强随机值
- [ ] **数据库密码**：PostgreSQL 和 Redis 密码已更换
- [ ] **HTTPS 配置**：SSL 证书已安装，强制 HTTPS
- [ ] **CORS 白名单**：已限制为生产域名
- [ ] **端口限制**：PostgreSQL/Redis 端口未暴露到公网
- [ ] **防火墙**：仅开放 80/443 端口
- [ ] **日志级别**：设置为 `info`，DEBUG=false
- [ ] **备份策略**：自动备份已启用
- [ ] **监控告警**：Prometheus + Grafana 已配置
- [ ] **速率限制**：已启用基于计划的限流

### 4.2 Docker Compose 生产配置

生产环境建议修改 `docker-compose.yml`：

```yaml
# 1. 移除 PostgreSQL 和 Redis 的端口映射
postgres:
  # 删除 ports 映射，仅容器内通信
  # ports:
  #   - "5432:5432"

redis:
  # 删除端口映射
  # ports:
  #   - "6379:6379"
  command: redis-server --appendonly yes --maxmemory 256mb --maxmemory-policy allkeys-lru --requirepass your-redis-password

# 2. 使用环境变量文件
server:
  env_file:
    - .env.production

# 3. 增加资源限制
server:
  deploy:
    resources:
      limits:
        cpus: "4.0"
        memory: 2G
```

### 4.3 SSL 证书配置（Let's Encrypt）

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

### 4.4 数据库备份策略

系统已内置自动备份服务（`db-backup`），每日自动备份，保留 7 天。

手动备份：

```bash
# Docker 环境
docker-compose exec postgres pg_dump -U saas_user vuemonitor > backup_$(date +%Y%m%d).sql

# 手动部署
pg_dump -U saas_user vuemonitor > backup_$(date +%Y%m%d).sql
```

恢复备份：

```bash
# Docker 环境
cat backup_20260512.sql | docker-compose exec -T postgres psql -U saas_user vuemonitor

# 手动部署
psql -U saas_user vuemonitor < backup_20260512.sql
```

### 4.5 监控配置

#### Prometheus

系统已内置 `/metrics` 端点。配置 Prometheus 采集：

  - job_name: 'xhs365'

    static_configs:
      - targets: ['localhost:8000']
```

#### Grafana

导入预置仪表盘：

```

## 7. 常见问题

### Q1: 服务启动失败，提示数据库连接错误

**原因**：PostgreSQL 未启动或密码不正确

**解决**：
```bash
# 检查 PostgreSQL 状态
docker-compose ps postgres

# 检查连接
docker-compose exec postgres pg_isready -U saas_user

# 检查密码
# 确认 .env 中 DB_PASSWORD 与 docker-compose.yml 一致
```

### Q2: Redis 连接超时

**原因**：Redis 未启动或密码不匹配

**解决**：
```bash
# 检查 Redis 状态
docker-compose exec redis redis-cli ping

# 如设置了密码
docker-compose exec redis redis-cli -a your-password ping
```

### Q3: 客户端无法连接服务端

**原因**：CORS 配置或网络问题

**解决**：
1. 检查 `.env` 中 `CORS_ORIGINS` 是否包含客户端地址
2. 检查防火墙是否开放 8000 端口
3. 检查服务端是否正常运行

### Q4: 数据库迁移失败

**原因**：迁移脚本冲突或数据库版本不一致

**解决**：
```bash
# 查看当前迁移版本
alembic current

# 查看迁移历史
alembic history

# 回退到指定版本
alembic downgrade <revision>

# 重新迁移
alembic upgrade head
```

### Q5: Docker 构建失败

**原因**：依赖下载超时或磁盘空间不足

**解决**：
```bash
# 清理 Docker 缓存
docker system prune -a

# 重新构建（无缓存）
docker-compose build --no-cache

# 检查磁盘空间
df -h
```

### Q6: 生产环境安全警告

**原因**：使用了默认密钥

**解决**：
1. 编辑 `.env` 文件
2. 使用 `python3 -c "import secrets; print(secrets.token_hex(32))"` 生成新密钥
3. 替换所有 `change-me` 开头的默认值
4. 重启服务：`docker-compose restart server`
