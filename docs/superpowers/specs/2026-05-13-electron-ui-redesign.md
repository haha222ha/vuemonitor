# XHS365 Electron 客户端 UI/UX 重设计方案

> 日期：2026-05-13 | 版本：V1.0

---

## 一、当前问题诊断

### 1.1 核心问题

| # | 问题 | 严重度 | 具体表现 |
|---|------|--------|---------|
| P1 | **无品牌识别度** | 🔴 | 登录页白卡+灰底，侧边栏深蓝#304156，像通用后台模板 |
| P2 | **视觉风格不统一** | 🔴 | Dashboard有渐变圆角卡片，AIView用inline style裸奔，MonitorView混搭 |
| P3 | **信息密度低** | 🟡 | 商品列表只显示ID和时间，缺少价格/销量等关键指标 |
| P4 | **侧边栏臃肿** | 🟡 | 9个菜单项平铺，采集调度/授权码等低频项占据核心位置 |
| P5 | **登录页简陋** | 🔴 | 纯白卡片+灰底，无品牌感，无产品介绍 |
| P6 | **AI分析展示差** | 🟡 | 分析结果用pre-wrap纯文本展示，无结构化渲染 |
| P7 | **无空状态设计** | 🟡 | 商品为空时只显示空表格，无引导 |
| P8 | **响应式不完善** | 🟢 | 移动端底部导航只有5项，缺失AI/对比等 |

### 1.2 当前代码结构

```
renderer/
├── App.vue              # 空壳 <router-view />
├── main.ts              # Element Plus 全量引入
├── layouts/MainLayout.vue  # 侧边栏+头部+内容区
├── views/               # 11个页面视图
├── components/          # 仅2个图表组件
├── stores/              # 9个Pinia store
├── composables/         # 快捷键
├── directives/          # 权限指令
├── i18n/                # 中英文
└── utils/api.ts         # axios封装
```

**关键发现**：components 目录几乎为空，所有 UI 逻辑都堆在 view 文件中（DashboardView 771行，SettingsView 1465行），没有可复用的组件体系。

## 二、设计语言

### 2.1 设计理念

**「数据驱动的专业工具感」** — 介于 SaaS 后台和消费级 App 之间，偏向专业工具但保持亲和力。

核心关键词：**清晰 · 高效 · 可信**

### 2.2 色彩系统

```
主色调：Indigo（靛蓝）— 专业、可信、科技感
  - Primary:    #4F46E5  (按钮、链接、活跃态)
  - Primary-L:  #818CF8  (Hover态)
  - Primary-D:  #3730A3  (Active态)

辅助色：
  - Success:    #10B981  (增长、在线、完成)
  - Warning:    #F59E0B  (告警、注意)
  - Danger:     #EF4444  (下跌、错误、删除)
  - Info:       #6B7280  (次要信息)

中性色：
  - BG-Page:    #F8FAFC  (页面背景)
  - BG-Card:    #FFFFFF  (卡片背景)
  - BG-Sidebar: #1E1B4B  (侧边栏深靛蓝)
  - Border:     #E2E8F0  (边框)
  - Text-1:     #0F172A  (主文字)
  - Text-2:     #475569  (次要文字)
  - Text-3:     #94A3B8  (辅助文字)

渐变：
  - Hero:       linear-gradient(135deg, #4F46E5, #7C3AED)
  - Success:    linear-gradient(135deg, #10B981, #34D399)
```

### 2.3 字体系统

```css
--font-sans: 'Inter', -apple-system, 'PingFang SC', 'Microsoft YaHei', sans-serif;
--font-mono: 'JetBrains Mono', 'Fira Code', 'Consolas', monospace;

--text-xs:   12px / 16px;   /* 辅助标签 */
--text-sm:   13px / 20px;   /* 次要内容 */
--text-base: 14px / 22px;   /* 正文 */
--text-lg:   16px / 24px;   /* 小标题 */
--text-xl:   20px / 28px;   /* 卡片标题 */
--text-2xl:  28px / 36px;   /* 页面标题 */
--text-3xl:  36px / 44px;   /* 大数字展示 */
```

### 2.4 间距与圆角

```
间距：4px 基准 → 4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48
圆角：
  - 小组件（Tag/Badge）: 6px
  - 输入框/按钮: 8px
  - 卡片: 12px
  - 弹窗: 16px
阴影：
  - 卡片: 0 1px 3px rgba(0,0,0,0.04), 0 1px 2px rgba(0,0,0,0.06)
  - 悬浮: 0 4px 16px rgba(0,0,0,0.08)
  - 弹窗: 0 8px 32px rgba(0,0,0,0.12)
```

## 三、布局重设计

### 3.1 整体布局

```
┌──────────────────────────────────────────────────────┐
│ ┌────────┐ ┌──────────────────────────────────────┐  │
│ │        │ │  顶部栏（搜索+通知+用户）             │  │
│ │  侧边栏 │ ├──────────────────────────────────────┤  │
│ │  可折叠 │ │                                      │  │
│ │        │ │  内容区                                │  │
│ │  Logo  │ │                                      │  │
│ │  ──── │ │                                      │  │
│ │  📊 盘面 │ │                                      │  │
│ │  📦 商品 │ │                                      │  │
│ │  🤖 AI  │ │                                      │  │
│ │  ──── │ │                                      │  │
│ │  ⏰ 调度 │ │                                      │  │
│ │  🔔 监控 │ │                                      │  │
│ │  ──── │ │                                      │  │
│ │  ⚙ 设置 │ │                                      │  │
│ │        │ │                                      │  │
│ │  ──── │ │                                      │  │

│ └────────┘ └──────────────────────────────────────┘  │

```

### 3.2 侧边栏改造

**当前**：220px 固定宽度，9项平铺，深蓝#304156
**改为**：

- 宽度：展开 240px / 折叠 64px，支持快捷键切换
- 背景：深靛蓝 #1E1B4B → #312E81 渐变
- Logo：XHS365 品牌字 + 折叠时只显示图标
- 分组：核心功能 / 工具 / 系统，用分割线区分
- 底部：套餐卡片（显示当前套餐+到期时间，引导升级）
- 活跃态：左侧4px主色竖条 + 背景微亮

菜单分组：

```
── 核心功能 ──
  📊 数据盘面    /dashboard
  📦 商品监控    /products
  🤖 AI 分析     /ai

── 工具 ──
  ⏰ 采集调度    /scheduler
  🔔 监控规则    /monitor
  📨 通知中心    /notifications

── 系统 ──
  ⚙ 设置        /settings
  🔑 授权管理    /license
```

### 3.3 顶部栏

**当前**：简陋的header，只有折叠按钮+通知+用户名
**改为**：

- 高度：56px
- 左侧：面包屑导航 / 页面标题
- 中间：全局搜索框（Cmd+K 唤起，搜索商品/规则/设置）
- 右侧：采集状态指示灯 + 通知铃铛 + 用户头像下拉

## 四、页面级重设计

### 4.1 登录页

**当前**：白卡片+灰底，零品牌感
**改为**：左右分栏布局

```
┌──────────────────────┬───────────────────────┐
│                      │                       │
│   品牌展示区          │   登录表单区            │
│                      │                       │
│   XHS365 Logo        │   欢迎回来             │
│   AI驱动的选品        │   ────────            │
│   决策系统            │   邮箱/昵称 [        ] │
│                      │   密码     [        ] │
│   ✦ 实时监控          │                       │
│   ✦ AI智能分析        │   [ 登录 ]            │
│   ✦ 趋势预警          │                       │
│                      │   没有账号？注册        │
│   渐变背景            │   白色背景             │
│   #4F46E5→#7C3AED   │                       │
└──────────────────────┴───────────────────────┘
```

- 左侧：品牌渐变背景 + Logo + 3个核心卖点 + 装饰性数据可视化插图
- 右侧：纯白背景，简洁表单，主色按钮
- 注册弹窗保持，但样式统一

### 4.2 数据盘面（Dashboard）

**当前**：4个统计卡+采集状态+风控告警+最近结果，功能完整但视觉层次不够
**改为**：

```
┌─────────────────────────────────────────────────┐
│  数据盘面                    上次刷新: 10:32  [刷新] │
├─────────┬─────────┬─────────┬───────────────────┤
│ 监控商品 │ 采集中  │ 队列任务 │ 定时任务           │
│   12    │   3     │   5     │   8               │
│ ↑2 本周 │ 运行中   │ 等待中   │ 已调度             │
├─────────┴─────────┴─────────┴───────────────────┤
│                                                   │
│  ┌─ 采集趋势 ──────────┐  ┌─ 风控状态 ─────────┐ │
│  │  📈 折线图           │  │  ✅ 安全            │ │
│  │  (最近24h采集量)      │  │  最近5条风控事件     │ │
│  └─────────────────────┘  └────────────────────┘ │
│                                                   │
│  ┌─ 最近采集结果 ──────────────────────────────┐  │
│  │  商品名 | 状态 | 数据变化 | 时间              │  │
│  └─────────────────────────────────────────────┘  │
└───────────────────────────────────────────────────┘
```

改进点：
- 统计卡片：大数字+小标签+趋势箭头，去掉渐变icon背景，改为纯色圆角图标
- 新增：采集趋势折线图（ECharts，最近24h）
- 采集状态卡：改为更紧凑的横向信息条
- 风控告警：增加安全/风险状态指示器
- 最近结果：增加「数据变化」列（如：销量+12%）

**改为**：卡片+列表双视图

- 报告导出保持，但UI统一

### 4.5 设置页

**当前**：1465行巨型单文件，左侧导航+右侧内容
**改为**：保持当前结构，但拆分为独立组件+统一样式

改进点：
- 拆分为：AccountSection / NotificationSection / SyncSection / CollectionSection / UpdateSection / DataSection / LogSection / ShortcutSection
- 统一 setting-item 组件：标题+描述+操作控件
- 去掉 inline style，全部用 CSS 类

## 五、组件体系

### 5.1 新增基础组件

| 组件 | 用途 | 替代当前 |
|------|------|---------|
| `StatCard.vue` | 统计数字卡片 | Dashboard 内联样式 |

| `SettingItem.vue` | 设置项统一布局 | SettingsView 内联 |

  --el-font-size-base: 14px;
  --el-bg-color: #F8FAFC;

}
