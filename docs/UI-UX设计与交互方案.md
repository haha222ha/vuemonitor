# XHS365 UI/UX 设计与交互方案

> 日期：2026-05-14 | 版本：V2.0 | 基于：产品重构方案V2.0

---

## 一、设计理念

### 1.1 核心定位

**「商业情报终端」** — 不是工具，是决策系统。

用户打开软件的第一反应应该是"今天有什么机会"，而不是"我要用什么功能"。

### 1.2 设计关键词

**洞察 · 决策 · 效率**

- **洞察**：数据可视化让趋势一目了然，排名百分位让竞争态势清晰
- **决策**：AI推荐+异动告警让用户知道该做什么，而非只看到数据
- **效率**：关键操作3步内完成，高频信息首屏可见

### 1.3 设计原则

| 原则 | 说明 | 反例 |
|-----|------|------|
| 信息先行 | 先展示结论，再展示数据 | 只展示原始数据让用户自己判断 |
| 渐进披露 | 首屏只展示核心，详情按需展开 | 首屏堆砌所有信息 |
| 真实驱动 | FOMO基于真实排名/基准数据 | 制造虚假稀缺或紧迫感 |
| 场景导航 | 按决策场景组织，不按工具功能组织 | "采集调度"是工具，"异动监控"是场景 |
| 一致性 | 同类信息用相同组件展示 | 不同页面用不同样式展示同类数据 |

## 二、信息架构

### 2.1 导航结构

```
机会雷达 (/dashboard)
  ├── 今日机会榜
  ├── 异动监控
  └── 品类热力图

我的商品 (/products)
  ├── 商品列表（带排名百分位）
  ├── 商品详情 (/products/:id)
  │   ├── 基本信息卡
  │   ├── 排名基准卡
  │   ├── 趋势图表
  │   └── AI分析入口
  └── 商品对比 (/compare)

AI决策 (/ai)
  ├── 快捷分析（Phase 1）
  ├── AI推荐（Phase 2，需服务端API）
  ├── 分析记录
  └── 分析报告（4种类型）

告警中心 (/monitor)
  ├── 告警规则管理
  ├── 告警事件列表
  └── 告警统计

通知中心 (/notifications)
  ├── 全部通知
  ├── 异动通知
  └── 系统通知

采集调度 (/scheduler)

设置 (/settings)

授权管理 (/license)
```

### 2.2 导航分组

```
── 洞察 ──
  🎯 机会雷达    /dashboard
  📦 我的商品    /products
  🤖 AI决策      /ai

── 管理 ──
  🔔 告警中心    /monitor
  📨 通知中心    /notifications  [未读数]
  ⏰ 采集调度    /scheduler

── 系统 ──
  ⚙ 设置        /settings
  🔑 授权管理    /license
```

#### 统计卡片交互

| 交互 | 行为 |
|-----|------|
| 卡片点击 | 跳转到对应区域（机会榜/异动/分析记录） |

#### 机会榜卡片交互

| 交互 | 行为 |
|-----|------|

|-----|------|

| 颜色编码 | 🔴critical / 🟡warning / 🟢info |

#### 品类热力图交互

| 交互 | 行为 |

## 四、全局交互规范

### 4.1 加载状态

| 场景 | 处理方式 |
|-----|---------|
| 页面首次加载 | 骨架屏(Skeleton)，保持布局结构 |
| 数据刷新 | 顶部进度条 + 刷新按钮loading |
| 按钮操作 | 按钮loading + 禁用 |
| AI分析 | 进度条 + 预计时间 + 取消按钮 |
| 采集任务 | 状态指示灯 + 进度百分比 |

### 4.2 空状态

| 场景 | 展示 |
|-----|------|
| 无商品 | 插图 + "添加第一个商品" + 引导按钮 |
| 无分析记录 | 插图 + "开始第一次AI分析" + 引导按钮 |
| 无告警 | ✅ "一切正常，暂无异动" |
| 无通知 | 📭 "暂无通知" |
| 搜索无结果 | 🔍 "未找到匹配结果，试试其他关键词" |
| **排名数据为空** | 📊 "数据积累中，添加更多商品后自动生成排名" + [添加商品] |
| **热力图无数据** | 🌡️ "需连接云端服务并积累数据" + [配置同步] |
| **基准数据不足** | 📏 "同类商品数据不足，暂无法生成基准" |
| **首页整体无数据** | 引导流程："1.添加商品 → 2.采集数据 → 3.查看洞察" |
| **离线状态** | 📡 "已离线，部分数据可能不是最新" + 灰色横幅 |

### 4.3 错误状态

| 场景 | 处理方式 |
|-----|---------|
| 网络断开 | 顶部红色横幅 "网络连接已断开" + 重试按钮 |
| API错误 | 卡片内错误提示 + 重试按钮 |
| AI分析失败 | 内联错误提示 + 重试按钮 + 切换模型选项 |
| 采集失败 | 通知 + 错误详情 + 重试按钮 |

### 4.4 实时更新

| 场景 | 处理方式 |
|-----|---------|
| WebSocket推送新通知 | 顶部弹出Toast + 通知铃铛数字+1 |
| 采集完成 | 商品卡片数据自动刷新 + 轻微闪烁动画 |
| AI分析完成 | 通知 + 分析记录自动出现 |
| 告警触发 | 顶部弹出Toast + 异动监控区自动更新 |

### 4.5 快捷键

| 快捷键 | 功能 |
|-------|------|
| `Cmd/Ctrl + K` | 全局搜索 |
| `Cmd/Ctrl + N` | 新建分析 |
| `Cmd/Ctrl + R` | 刷新当前页数据 |
| `Cmd/Ctrl + [` | 侧边栏折叠/展开 |
| `1-9` | 快速切换导航（侧边栏折叠时） |

### 4.6 动画规范

| 动画 | 时长 | 缓动函数 | 场景 |
|-----|------|---------|------|
