### 阶段1：安全基线加固 [P0-紧急]

**目标**：修复审计报告中的高危安全问题，确保生产环境安全基线

| # | 任务 | 说明 | 状态 |
|---|------|------|------|
| 1.1 | 更换默认密钥 | JWT_SECRET/JWT_REFRESH_SECRET/ENCRYPTION_KEY 使用强随机值 | ✅ 本地已更换 |
| 1.2 | 更换数据库密码 | PostgreSQL saas_pass → 强密码，同步更新 .env 和服务配置 | pending(需服务器操作) |
| 1.3 | Redis密码配置 | 生产环境Redis设置密码 | pending(需服务器操作) |
| 1.4 | CORS白名单收紧 | 限制为生产域名，移除localhost | ✅ 已完成 |
| 1.5 | 数据库端口限制 | 移除PostgreSQL/Redis宿主机端口映射 | pending(需服务器操作) |
| 1.6 | 环境变量管理 | 敏感配置从docker-compose.yml移至.env或Vault | ✅ 本地已加固 |

> ✅ **HTTPS已解决**：Cloudflare Flexible SSL + 始终HTTPS重定向，用户端流量全程HTTPS，无需额外配置源站SSL。

**验收标准**：审计报告中所有🔴高危项全部修复（SEC-4 HTTPS已由CF解决）
